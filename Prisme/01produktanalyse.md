# 01 — Produktanalyse: Brilliant.org og MatematikFessor

Analysen bygger udelukkende på **offentligt tilgængelig information** (produktsider, hjælpecentre, priser, presseomtale) og på generel viden om, hvordan platformene fremstår. Formålet er at forstå *mekanikken* i stærke digitale læringsplatforme — ikke at kopiere indhold, branding eller proprietære systemer.

---

## 1.1 Brilliant.org

**Hvad det er:** En amerikansk, abonnementsbaseret STEM-læringsplatform (matematik, fysik, datalogi, "data & analyse", kvantitativ finans m.m.) med >100 kurser. Kernefilosofi: *"Learn by doing"* — man lærer ved at manipulere, gætte og løse, ikke ved at læse.

| Dimension | Sådan fremstår Brilliant |
|-----------|--------------------------|
| **Brugergrupper** | Primært selvlærende voksne, nysgerrige teenagere, professionelle der genopfrisker/opgraderer STEM. **Ikke** klassebaseret; intet lærer- eller skolelag. |
| **Elev-/brugeroplevelse** | Korte "sessions" (5–15 min), meget lav tekstmængde, høj interaktivitet. Man trækker, klikker, justerer skydere og *ser* konsekvensen med det samme. Progression føles som at løse gåder. |
| **Lærer-oplevelse** | Reelt fraværende som produktlag. Ingen klasser, afleveringer eller lærerdashboard. (Enkelte skoletilbud har eksisteret, men kerneproduktet er B2C.) |
| **Lektionsdesign** | En lektion = en sekvens af interaktive "frames". Hvert frame stiller ét lille spørgsmål eller én manipulation, giver øjeblikkelig respons og bygger intuition *før* formalismen. Formler introduceres sent. |
| **Opgavedesign** | Konceptuelle, ofte visuelle problemer der tvinger ræsonnement frem for udenadslære. Distraktorer er designet til at ramme typiske fejltanker. |
| **Auto-retning** | Øjeblikkelig. Interaktive svarformater (klik på figur, træk punkt, vælg, indtast tal). Feedback kommer instant. |
| **Feedback** | Forklarende: efter et svar vises *hvorfor*. En indbygget AI-tutor ("Koji") kan guide skridt-for-skridt uden at afsløre svaret. |
| **Hints** | Progressive: man kan bede om et hint, se en forklaring, eller lade tutoren stille et ledende spørgsmål. |
| **Gamification** | Streaks (dage i træk), daglige udfordringer, XP-lignende fremdrift, "levels". Nedtonet og elegant — ikke støjende badges. |
| **Fremskridtsopfølgning** | Personlig sti/algoritme der foreslår næste kursus; procent gennemført pr. kursus; streak-kalender. Individuel, ikke klassebaseret. |
| **Afleveringsværktøjer** | Ingen (ingen lærer der uddeler opgaver). |
| **Dashboards** | Personligt fremdrifts-overblik. Intet klasse- eller institutionsdashboard. |
| **Indholdsstruktur** | Kurser → lektioner → interaktive frames. Kuraterede "paths" på tværs af kurser. |
| **Forretningsmodel** | Freemium → premium-abonnement (individ/familie), månedlig ca. 20–28 USD, årligt ca. 120–150 USD. Gratis "preview" + daglig udfordring. |

**Styrker:** Uovertruffen på *begribelighed* og *lyst*. Visuel intuition, lav friktion, høj kvalitet, æstetik i topklasse. Får svært stof til at føles legende.

**Svagheder (set fra en dansk skole):** Engelsksproget; forbrugerpris pr. individ (ikke skoleindkøb); **intet lærer-/klasselag** (ingen afleveringer, ingen holdopfølgning); ingen kobling til Fælles Mål eller danske prøver; mængdetræning og "drilling" er ikke fokus; ingen differentiering styret af en lærer; ingen forældre-/skoleadministration.

## 1.2 MatematikFessor (EduLab, i dag under Alinea/Egmont)

**Hvad det er:** Danmarks mest udbredte online matematikportal for **0.–10. klasse**. Bygget til den danske folkeskole med opgaver, videoer, forløb og et fuldt lærerunivers.

| Dimension | Sådan fremstår MatematikFessor |
|-----------|-------------------------------|
| **Brugergrupper** | Elever (0.–10. kl.), lærere, og skole/kommune som indkøber. Klare klassebaserede roller. |
| **Elev-oplevelse** | Opgavesæt og træningsforløb med **automatisk retning** og **videohjælp til hvert spørgsmål**. Fokus på mængde, repetition og at "komme igennem" et emne. Fungerer også til print/aktiviteter i klassen. |
| **Lærer-oplevelse** | Bibliotek med forløb, lektioner, opgavesæt, aktiviteter og arbejdsark, som kan **filtreres** (emne, indholdstype, klassetrin) og **tildeles** elever. **Evaluering** i topmenuen med elevresultater og livestatistik. |
| **Lektionsdesign** | Emneopdelt efter klassetrin og matematiske områder (tal, geometri, statistik osv.). Videoer forklarer, opgaver træner. Mere "forklar → træn" end "opdag selv". |
| **Opgavedesign** | Store mængder opgaver pr. emne, ofte parametriske/genererede, med stigende sværhed. Auto-rettbare formater. |
| **Auto-retning** | Retter automatisk og giver **øjeblikkeligt resultat**. Kernen i produktet. |
| **Feedback** | Rigtigt/forkert + adgang til forklarende **video** pr. opgave. |
| **Hints** | Primært via videoforklaring og gennemgange snarere end en trinvis hint-stige i selve opgaven. |
| **Gamification** | Point/belønning og fremdriftsfølelse; mere afdæmpet end en ren spilmekanik. |
| **Fremskridtsopfølgning** | Klassestatistik og individuelle resultater; læreren følger med live. |
| **Afleveringsværktøjer** | Stærke: tildel forløb/opgavesæt til hele hold eller enkeltelever, med differentiering. |
| **Dashboards** | Fuldt **lærerdashboard** (resultater, statistik, evaluering) + elevoverblik. |
| **Indholdsstruktur** | Fag → klassetrin → emner → forløb/lektioner/opgavesæt/aktiviteter/arbejdsark. |
| **Differentiering** | Læreren kan skrue **klassetrinsniveauet op/ned pr. elev** — nem, konkret differentiering. |
| **Forretningsmodel** | B2B/B2G: skole- og kommunelicenser (typisk pr. elev/år eller skolelicens), ofte via kommunalt indkøb; en del af Alineas læremiddelunivers. |

**Styrker:** Perfekt tilpasset **dansk folkeskolepraksis**: lærerens workflow (tildel, følg op, evaluér), auto-retning i stor skala, differentiering pr. elev, videohjælp, print-materialer, bred emnedækning. Lav friktion for travle lærere.

**Svagheder (mulighed for os):** Primært **matematik** (ikke fysik/kemi/biologi/geografi samlet). Oplevelsen er mere *træning* end *begrebsmæssig aha*: mindre af Brilliants legende, manipulerbare interaktivitet. Videoer bærer forklaringen frem for indbyggede interaktive modeller. UI'et bærer år på bagen. Begrænset adaptiv/mastery-baseret motor sammenlignet med state of the art.

## 1.3 Syntese — hvad de to lærer os

```
        BRILLIANT                         MATEMATIKFESSOR
   (forstå · visuelt · leg)          (træn · ret · lærerstyr)
            \                                   /
             \                                 /
              \                               /
               ▼                             ▼
        ┌───────────────────────────────────────────┐
        │                 PRISME                     │
        │  Interaktiv begrebslæring  +  mængdetræning │
        │  + auto-retning + lærerdashboard            │
        │  + Fælles Mål + naturfagene samlet + GDPR   │
        └───────────────────────────────────────────┘
```

**Ti principper vi tager med:**

1. **Vis før du fortæller** (Brilliant): en interaktiv model før formlen.
2. **Kort tekst, høj interaktivitet** — hver skærm har én ting man kan gøre.
3. **Øjeblikkelig, forklarende feedback** — ikke bare "forkert".
4. **Trinvise hints** der guider uden at afsløre.
5. **Auto-retning i stor skala** (MatematikFessor) som fundament for lærerens tidsbesparelse.
6. **Genererede, parametriske opgaver** så træning aldrig slipper op.
7. **Lærerstyret tildeling og differentiering pr. elev.**
8. **Live klassestatistik** som lærerens overblik.
9. **Afdæmpet, meningsfuld gamification** (streaks/mestring), ikke støj.
10. **Én ramme, mange fag** — samme motor genbruges på tværs af de fem fag.
